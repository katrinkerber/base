Separation of concerns

JavaScript behaviour should never be linked to classes. By using classes as styling hooks only and the data-role for JS behaviour only, we are preventing any possibility of the two ever crossing streams. Any changes to classes will never affect the JS behaviour and vice versa. This post explains this concept and its advantages well.

File organization


js/
|
|-- libs/         # Includes any libraries and plugins
|-- global.js    # Global site functions, Mixture imports plugins and minifies into global.min.js
|
|-- # Add more JS files as required for specific sections of the site, i.e. 'studio.js'

Syntax

  • Use soft-tabs set to 2 spaces.
  • Nested elements should be indented once (2 spaces).
  • Always use single quotes for selectors.

/* HTML */
<button data-role="slidein-toggle">Toggle me</button>

/* JS */
function slideInContent() {
  var $slideToggleBtn = $('[data-role="slidein-toggle"]');
  ...
}
